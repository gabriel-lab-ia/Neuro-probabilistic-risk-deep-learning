# Security Policy

## Scope

This repository is a research prototype and not a production medical system. Security issues should still be reported responsibly, especially if they affect:

- dependency integrity,
- unsafe artifact publication,
- accidental exposure of local secrets or environment data,
- misleading presentation of model confidence or risk outputs.

## Reporting

Please report security-sensitive issues privately to the repository maintainers before opening a public issue.

When reporting, include:

- affected file or workflow,
- reproduction steps,
- potential impact,
- suggested mitigation if available.

## Additional safety note

Because the repository handles neurological risk modeling concepts, issues that may cause unsafe overstatement of certainty or misuse of predictions should also be treated as high-priority responsible disclosure items.
